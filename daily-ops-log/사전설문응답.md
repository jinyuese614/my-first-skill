# 사전설문 응답 기록

**일시**: 2026-02-05

## Q1. 지금 회사에서 어떤 일을 맡고 계세요?
**A1.** User Growth PM을 맡고 있음. 론칭할 제품에 Creator이든 User이든 많이 들어올 수 있도록 전략과 운영을 도맡는 직무.

## Q2. 하루가 어떻게 흘러가나요? 어떤 툴을 많이 쓰시나요?
**A2.** 지금은 Social Test라는 프로젝트를 가장 비중 있게 진행 중. 아침에 출근하면 유튜브, 틱톡, 인스타그램 채널에 영상 업로드하고, 채널 성과 분석하고, 바이브코딩으로 대시보드 개선하고, 채널 운영하면서 본 성과나 인사이트를 정리하는 운영일지도 작성. 가장 많이 쓰는 툴은 유튜브 스튜디오, 틱톡 스튜디오, 인스타그램 대시보드, 내부 콘텐츠 자동화 사이트.

## Q3. 반복적이고 시간이 오래 걸리는 작업이 있나요?
**A3.** 같은 영상을 채널마다 하나씩 올리는 것이 번거롭고, 성과 숫자 뽑아서 분석하는 게 반복적이고 시간이 오래 걸림.

## Q4. 영상 업로드는 얼마나 자주, 시간은 얼마나 걸리나요?
**A4.** AI 크리에이터 3-4개를 가지고 유튜브, 틱톡, 인스타그램에 모두 올리고 있어서 9-12개 채널 운영 중. 새 영상 올릴 때 늘 ×3을 해야 해서, 한 크리에이터당 15-30분 소요. 영상 파일은 내부 콘텐츠 매니지먼트 툴에서 가져와서 유튜브 → 틱톡 → 인스타그램 순서로 업로드.

## Q5. 성과 분석은 어떻게 하고 계세요?
**A5.** 바이브코딩으로 유튜브 API, 틱톡 API, 인스타그램 API 연동한 성과 분석 툴을 만들었으나, 동기화 시간이 느리고 인스타그램 API 오류가 자주 발생해서 현재는 일일이 각 채널 스튜디오에서 직접 확인하고 있음. 운영일지는 Lark 문서에 정리.

## Q6. 운영일지에는 어떤 내용을 쓰세요?
**A6.** 주로 오늘 어떤 영상을 올렸고, 오늘 하루 동안의 성과는 어떠했고, 그 성과를 기반으로 콘텐츠마다의 장단점 등도 같이 작성.

## Q7. 결과물은 어디에 저장/공유하세요?
**A7.** Lark 문서에 운영일지로 정리. Claude가 초안을 만들어주면 Lark에 붙여넣기 하는 방식으로 충분.
